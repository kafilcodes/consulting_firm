## üåü Overview

<p align="center">
  <img src="https://firebasestorage.googleapis.com/v0/b/github-readme-images.appspot.com/o/consulting-banner.png?alt=media" alt="SKS Consulting Banner" width="100%"/>

SKS Consulting is a comprehensive enterprise-grade business consulting platform that connects businesses with professional consulting services. The platform serves as a complete ERP system for consulting firms, offering role-based dashboards and workflows for clients, employees, consultants, and administrators.

The application streamlines the entire consulting process from service discovery and order management to communication, document exchange, and payment processing - all within a secure, intuitive interface.

## üì¶ Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/sks-consulting.git
   cd sks-consulting
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   ```bash
   cp .env.example .env.local
   ```
   Fill in your environment variables in `.env.local`:
   ```
   # Firebase Configuration
   NEXT_PUBLIC_FIREBASE_API_KEY=
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=
   NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
   NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
   NEXT_PUBLIC_FIREBASE_APP_ID=
   
   # Razorpay Configuration
   NEXT_PUBLIC_RAZORPAY_KEY_ID=
   RAZORPAY_KEY_SECRET=
   
   # Email Configuration
   EMAIL_SERVICE=
   EMAIL_USER=
   EMAIL_PASSWORD=
   ```

4. **Run the development server**
   ```bash
   npm run dev
   ```

5. **Build for production**
   ```bash
   npm run build
   npm start
   ```

## üíª Usage

After installation, the application will be accessible at `http://localhost:3000`. Users can:

1. Register or log in using email/password or Google authentication
2. Access role-specific dashboards based on their assigned role
3. Browse services and create orders (clients)
4. Manage assigned orders and communicate with clients (employees/consultants)
5. Oversee all operations and manage users (admins)

## üîß Configuration

### Firebase Setup

1. Create a Firebase project at [https://console.firebase.google.com/](https://console.firebase.google.com/)
2. Enable Authentication (Email/Password and Google providers)
3. Create a Firestore database
4. Set up Storage rules
5. Configure Firebase Hosting
6. Add your Firebase configuration to `.env.local`

### Razorpay Setup

1. Create a Razorpay account at [https://razorpay.com/](https://razorpay.com/)
2. Obtain API keys from the dashboard
3. Add Razorpay configuration to `.env.local`

### Security Rules

Example Firestore security rules:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // User profiles can be read by the user themselves or admins
    match /users/{userId} {
      allow read: if request.auth.uid == userId || 
                    get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      allow write: if request.auth.uid == userId || 
                     get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    // Orders can be read by the client who created it, the assigned employee, or admins
    match /orders/{orderId} {
      allow read: if request.auth.uid == resource.data.clientId || 
                    request.auth.uid == resource.data.assignedEmployeeId || 
                    get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      allow create: if request.auth != null;
      allow update: if request.auth.uid == resource.data.clientId || 
                      request.auth.uid == resource.data.assignedEmployeeId || 
                      get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
```

## üìä System Architecture

<p align="center">
  <img src="https://firebasestorage.googleapis.com/v0/b/github-readme-images.appspot.com/o/architecture-diagram.png?alt=media" alt="System Architecture" width="80%"/>
</p>

The SKS Consulting Platform follows a modern, serverless architecture:

1. **Client Application (Next.js)**: The frontend application built with Next.js and React
2. **Authentication (Firebase Auth)**: Handles user authentication and session management
3. **Database (Firestore)**: Stores application data in a NoSQL document database
4. **Storage (Firebase Storage)**: Manages file uploads and secure document storage
5. **Cloud Functions**: Handles backend business logic, notifications, and integrations
6. **Payment Gateway (Razorpay)**: Processes payments securely

## üîÑ Workflow

The typical workflow in the application:

1. Client browses available services
2. Client creates an order and uploads required documents
3. Admin assigns the order to an employee/consultant
4. Employee accesses the order and begins work
5. Client and employee communicate through the chat system
6. Employee completes the work and uploads deliverables
7. Client reviews and approves the completed work
8. Payment is processed through Razorpay
9. Order is marked complete
